    Configuração Git

    Configurar o Git
    git config --global user.name "[name]"

git config --global user.email "[email]"

    Criar chave SSH

    No terminal:

    Criar chave SSH
    ssh-keygen -t ed25519 -C "exemplo@academiafa.edu.pt"

    Pergunta o caminho desejado para guardar a chave. Ao clicar ENTER a chave é salva no caminho default.

    Introduzir e confirmar uma palavra passe.

    Iniciar ssh-agent em segundo plano
    eval "$(ssh-agent -s)"
    Adicionar a chave SSH ao ssh-agent
    ssh-add ~/.ssh/id_ed25519
    Copiar a chave SSH
    cat ~/.ssh/id_ed25519.pub

    Copiar a chave que aparece no terminal.

    Exemplo de chave SSH: ssh-ed25519 ABCDEFDKvCSnKTPLH9Vntk/wfF9CX_chave_exemplo

    No GitHub ir até às settings > SSH and GPG keys.

    3. Clone do repositório remoto

    Uma vez que o Git já está configurado para a nossa máquina, podemos clonar um repositório do GitHub.

    Clonar um repositório
    git clone git@github.com:exemplo/workshop_git.git

    4. Enviar ficheiros para a repo

    Vamos agora adicionar alguns ficheiros.

    Para adicionar os ficheiros ao repositorio local usa-se o git add . .

    Agora temos de criar um "commit" com uma certa mensagem. Este "commit" vai ser a "versão" do nosso repositório.

    git commit -m "<a vossa mensagem>"

    Para enviar os ficheiros para o repositório remoto: git push. Podem ter que introduzir git push --set-upstream origin master.

    Os vossos ficheiros devem estar agora disponiveis no GitHub.

    8. Reverter para versões anteriores

    Caso queiram voltar a uma versão anterior do vosso repositório, podem faze-lo assim:

    Reverter para um commit antigo
    git reset --hard [SHA]

    Atenção: todos os commits mais recentes que esta versão vão ser apagados!
